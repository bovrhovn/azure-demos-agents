Microsoft Agent Framework

A next-generation, open-source framework for building AI-powered agents and multi-agent workflows — the direct successor to both Semantic Kernel and AutoGen.

What is Agent Framework?

Microsoft Agent Framework is a modern SDK for building AI applications using agents and graph-based workflows. It combines the simplicity of AutoGen's agent abstractions with the enterprise-grade features of Semantic Kernel — session state management, type safety, middleware, and telemetry — and adds explicit multi-agent orchestration on top.

Agents can connect to Azure AI Foundry, Azure OpenAI, OpenAI, Anthropic, Ollama, and more. Workflows offer checkpointing and human-in-the-loop support for complex, long-running tasks.

Two Primary Capabilities

🤖 Agents

Individual agents that use LLMs to process inputs, call tools and MCP servers, and generate responses. Supports multiple providers including Azure OpenAI, OpenAI, Anthropic, and Ollama.

🔗 Workflows

Graph-based workflows that connect agents and functions for multi-step tasks. Provides type-safe routing, checkpointing, and human-in-the-loop support for complex orchestration scenarios.

Agents vs. Workflows — When to Use Which

Use an Agent when… Use a Workflow when…
The task is open-ended or conversational The process has well-defined steps
You need autonomous tool use and planning You need explicit control over execution order
A single LLM call (possibly with tools) suffices Multiple agents or functions must coordinate

💡 If you can write a plain function to handle the task, do that instead of using an AI agent.

Quick Start (C#)

Install the NuGet package and spin up your first agent in just a few lines:

dotnet add package Microsoft.Agents.AI.Foundry --prerelease

AIAgent agent = new AIProjectClient(
        new Uri("https://your-foundry-service..."),
        new AzureCliCredential())
    .AsAIAgent(
        model: "gpt-5.4-mini",
        instructions: "You are a friendly assistant.");

Console.WriteLine(await agent.RunAsync("What is the largest city in France?"));

Why Agent Framework?

Semantic Kernel and AutoGen pioneered AI agents and multi-agent orchestration at Microsoft. Agent Framework is the next generation of both — built by the same teams.

  • Merges AutoGen's simple single- and multi-agent abstractions with Semantic Kernel's enterprise features
  • Adds graph-based workflows for explicit, deterministic multi-agent execution paths
  • Provides robust state management for long-running and human-in-the-loop scenarios
  • Open-source and welcomes community contributions
  • Migration guides available for both Semantic Kernel and AutoGen users
